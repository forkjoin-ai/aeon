## Stable Throughput Witness `öÝ`

The current route is:

1. Form `öÝ_spec := JacksonTrafficData.spectralThroughput`, the resolvent candidate `ö£ (I - P)^{-1}` under `spectralRadius P < 1`.
2. Prove the traffic equations with `spectralThroughput_fixed_point`, so `öÝ_spec` is a genuine fixed point.
3. Discharge the still-external side conditions `0 <= öÝ_spec i` and `öÝ_spec i < ö¥_i`.
4. Use `JacksonTrafficData.spectralNetworkData` as the direct product-form witness.
5. Promote the same candidate to the monotone least-fixed-point path with the Knaster-Tarski-style dominance bridge `constructiveThroughput_le_of_real_fixed_point`, specialized by `constructiveThroughput_le_spectralThroughput`, and conclude finiteness/stability via `constructiveThroughput_finite_of_spectral`, `constructiveThroughput_stable_of_spectral`, and `constructiveNetworkDataOfSpectral`.

`JacksonExactClosure.lean` now packages the exact side of that story explicitly: once `spectralRadius P < 1` is known, any nonnegative stable real fixed point of the traffic equations is unique, equals the constructive least fixed point after `toReal`, and yields the same constructive network witness, mean occupancy, and `lintegral` balance laws without going through `throughputEnvelopeApprox n`.

`JacksonFeedForwardClosure.lean` now isolates one nontrivial raw exact subclass of that route: in the bounded two-node feed-forward ceiling witness coming from `StateDependentQueueFamilies.TwoNodeAdaptiveRoutingParameters`, the routing matrix squares to zero, so the explicit throughput candidate already equals the constructive least fixed point and the same product-form mean-occupancy / `lintegral` laws follow directly from the raw parameters.

## Adaptive Stability Transfer

The adaptive extension is now split into two layers.

1. `JacksonQueueing.AdaptiveJacksonTrafficData` mechanizes the comparison principle: state-indexed routing steps are bounded by any dominating Jackson kernel, and the `supremumKernel` specialization recovers the user-facing ceiling construction when that pointwise ceiling is still substochastic.
2. `Axioms.AdaptiveSupremumStabilityAssumptions` packages the remaining Lyapunov step as explicit synthesis data: the adaptive expected Lyapunov is bounded by a ceiling expected Lyapunov from raw domination data, and the shell now derives the drift-gap lower bound either from an automatically synthesized minimum-slack selector, from raw nonnegative scores that it normalizes internally, from arbitrary real scores after positive-part normalization, from an explicit selector-based one-hot decomposition, from a normalized nonnegative weighted decomposition with pointwise node slack bounds, from raw service slack, or from raw routing pressure. At that point the existing state-dependent stability and terminal-balance schemas apply, and positive recurrence and stationary-law existence are derived by the base schema from explicit bridge/drift/small-set witnesses instead of being supplied as free props.
3. `StateDependentQueueFamilies.TwoNodeAdaptiveRoutingParameters` now closes one concrete raw adaptive family end to end: from nonnegative arrivals, a reroute probability `0 <= p < 1`, and nodewise service slack, it constructs the dominating ceiling kernel, proves the ceiling's strict row-substochastic spectral side conditions, bounds the adaptive constructive throughput by a closed-form candidate, derives a linear drift witness on the bounded adaptive state space, and bridges that witness into the generic stationary/terminal balance schema with only the honest residual recurrence assumptions left explicit.
4. `AdaptiveWitnesses.lean` now exports that same concrete raw adaptive family as a machine-readable witness catalog, so the bounded two-node adaptive `öÝ` closure is consumed by runtime artifact checks instead of remaining only an internal theorem package detail.

The open adaptive part is now narrower: automatically discovering richer chosen-Lyapunov decompositions for arbitrary adaptive Lyapunov choices, rather than supplying one of the built-in minimum-slack, normalized-score, positive-part-score, explicit-selector, normalized-weighted, raw service-slack, or raw routing-pressure forms.
On the Jackson side, the boundary is narrower too: the package now has an exact fixed-point closure under `spectralRadius P < 1`, a raw sufficient criterion built from `maxExternalArrival` and `maxIncomingRoutingMass`, organizes the current witness family as a finite-step ladder `throughputEnvelopeApprox n`, where `n = 0` is the global envelope, `n = 1` is `localThroughputEnvelope`, and `n = 2` is `secondOrderThroughputEnvelope`, proves the ladder is descending, certifies its gap to `öÝ_spec` by explicit scalar residual terms, adds a routing-shaped node-local residual ladder `throughputResidualApprox n`, accepts direct service bounds against that ladder, packages the `minServiceRate` corollary as an automatic witness constructor, and now also turns any certified ladder stage into a direct finite-network product-form and `lintegral` closure through `JacksonEnvelopeClosure.lean`. What remains open there is deriving exact fixed points or still-sharper arrival/routing certificates directly from raw `(ö£, P, ö¥)` beyond that current exact-witness-plus-ladder/residual scaffold.
